# Governance\n\n## Definition of Done (DoD)\n\nA user story, task, or bug fix is considered Done when it meets the following criteria:\n\n1.  **Code Complete:** All required code has been written and is syntactically correct.\n2.  **Unit Tests Pass:** All relevant unit tests are written and passing.\n3.  **Integration Tests Pass:** If applicable, integration tests pass.\n4.  **Acceptance Criteria Met:** All acceptance criteria defined for the story are satisfied.\n5.  **Two-Eye Review:** The code has undergone a thorough code review by at least one other developer.\n6.  **Documentation Updated:** All necessary documentation (e.g., READMEs, inline comments, API docs) has been updated.\n7.  **No regressions:** Existing functionality remains unaffected.\n8.  **Deployed to Staging (if applicable):** The change has been successfully deployed to a staging environment for final testing.\n\n## Two-Eye Review Policy\n\nEvery code change (feature, bug fix, refactor) must be reviewed by at least one other developer before it can be merged into the main branch.\n\nThe reviewer should:\n- Verify the code meets the Definition of Done.\n- Check for code quality, maintainability, and adherence to coding standards.\n- Ensure the solution is appropriate for the problem.\n- Provide constructive feedback.\n- Approve the changes once satisfied, or request necessary modifications.\n\nThe author is responsible for addressing review feedback and ensuring the code is ready for merging.
